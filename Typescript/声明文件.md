https://www.typescriptlang.org/dt/search

## 发布

声明文件发布到 npm 的主要方法有两种:

- 与您的 npm 软件包捆绑在一起
- 在 npm 上发布到@types

```json
// package.json
{
  "name": "awesome",
  "author": "Vandelay Industries",
  "version": "1.0.0",
  "main": "./lib/main.js",
  //   请注意，该"typings"字段与相同"types"，也可以使用
  "types": "./lib/main.d.ts"
}
```

## 声明参考 （自已手动编写）

- 只有底层库代码时，需要编写声明文件

### 具有属性的对象

```ts
let result = myLib.makeGreeting('hello, world');
console.log('The computed greeting is:' + result);
let count = myLib.numberOfGreetings;
```

```ts
// 使用declare namespace描述由点号访问类型或值
declare namespace myLib {
  function makeGreeting(s: string): string;
  let numberOfGreetings: number;
}
```

### 重载函数

```ts
let x: Widget = getWidget(43);
let arr: Widget[] = getWidget('all of them');
```

```ts
declare function getWidget(n: number): Widget;
declare function getWidget(s: string): Widget[];
```

### 可重用类型 (Interfaces)

```ts
greet({
  greeting: 'hello world',
  duration: 4000,
});
```

```ts
// 使用 interface 定义具有属性的类型
interface GreetingSettings {
  greeting: string;
  duration?: number;
  color?: string;
}
declare function greet(setting: GreetingSettings): void;
```

### 可重用类型 (Type Aliases)

```ts
function getGreeting() {
  return 'howdy';
}
class MyGreeter extends Greeter {}
greet('hello');
greet(getGreeting);
greet(new MyGreeter());
```

```ts
// 使用类型别名为类型进行速记
type GreetingLike = string | (() => string) | MyGreeter;
declare function greet(g: GreetingLike): void;
```

### 组织类型

```ts
const g = new Greeter('Hello');
g.log({ verbose: true });
g.alert({ modal: false, title: 'Current Greeting' });
```

```ts
// 使用namespace来组织类型
declare namespace GreetingLib {
  interface LogOptions {
    verbose?: boolean;
  }
  interface AlertOptions {
    modal: boolean;
    title?: string;
    color?: string;
  }
}
```

```ts
// 您还可以在一个声明中创建嵌套命名空间
declare namespace GreetingLib.Options {
  // Refer to via GreetingLib.Options.Log
  interface Log {
    verbose?: boolean;
  }
  interface Alert {
    modal: boolean;
    title?: string;
    color?: string;
  }
}
```

### Classes

```ts
const myGreeter = new Greeter('hello, world');
```


## 库结构

- 在 .d.ts 文件中使用 export default 需要 esModuleInterop: true 才能工作

```ts
// module-function.d.ts
const x = require('foo');
// Note: calling 'x' as a function
const y = x(42);
```

```ts
// module-class.d.ts
const x = require('bar');
// Note: using 'new' operator on the imported variable
const y = new x('hello');
```

```ts
// module-plugin.d.ts
const jest = require('jest');
require('jest-matchers-files');
```

### Modules.d.ts

```ts
// 模块代码中的命名空间
// This represents the JavaScript class which would be available at runtime
export class API {
  constructor(baseURL: string);
  getInfo(opts: API.InfoRequest): API.InfoResponse;
}
// This namespace is merged with the API class and allows for consumers, and this file
// to have types which are nested away in their own sections.
declare namespace API {
  export interface InfoRequest {
    id: string;
  }
  export interface InfoResponse {
    width: number;
    height: number;
  }
}
```

```ts
// 在 .d.ts 文件中使用 export default 需要 esModuleInterop: true 才能工作。
declare function getArrayLength(arr: any[]): number;
declare namespace getArrayLength {
  declare const maxInterval: 12;
}
export = getArrayLength;
```

```ts
// 使用 export as namespace 来声明您的模块将在 UMD 上下文中的全局范围内可用
export as namespace moduleName;
```

## 注意

```ts
不要使用：Number, String, Boolean, Symbol, or Object原始数据类型
/* WRONG */
function reverse(s: String): String;

/* OK */
function reverse(s: string): string;
```

```ts
不要将返回类型 any 用于其值将被忽略的回调

/* WRONG */
function fn(x: () => any) {
  x();
}

/* OK */
function fn(x: () => void) {
  x();
}
```


```ts
不要在回调中使用可选参数
/* WRONG */
interface Fetcher {
  getObject(done: (data: any, elapsedTime?: number) => void): void;
}
```

```ts
不要编写仅在回调数量上有所不同的单独重载
/* WRONG */
declare function beforeAll(action: () => void, timeout?: number): void;
declare function beforeAll(
  action: (done: DoneFn) => void,
  timeout?: number
): void;


/* OK */
declare function beforeAll(
  action: (done: DoneFn) => void,
  timeout?: number
): void;
```

```ts
不要在更具体的重载之前放置更一般的重载
/* WRONG */
declare function fn(x: any): any;
declare function fn(x: HTMLElement): number;
declare function fn(x: HTMLDivElement): string;
var myElem: HTMLDivElement;
var x = fn(myElem); // x: any, wat?

通过将更一般的签名放在更具体的签名之后来对重载进行排序
/* OK */
declare function fn(x: HTMLDivElement): string;
declare function fn(x: HTMLElement): number;
declare function fn(x: any): any;
var myElem: HTMLDivElement;
var x = fn(myElem); // x: string, :)
```

```ts
不要写几个只在尾随参数上不同的重载
/* WRONG */
interface Example {
  diff(one: string): number;
  diff(one: string, two: string): number;
  diff(one: string, two: string, three: boolean): number;
}

/* OK */
interface Example {
  diff(one: string, two?: string, three?: boolean): number;
}
```

```ts
不要编写仅在一个参数位置上因类型不同而不同的重载
/* WRONG */
interface Moment {
  utcOffset(): number;
  utcOffset(b: number): Moment;
  utcOffset(b: string): Moment;
}

/* OK */
interface Moment {
  utcOffset(): number;
  utcOffset(b: number | string): Moment;
}
```

## 深入

### Types

* 类型别名声明 (type sn = number | string;)
* 接口声明 (interface I { x: number[]; })
* 类声明  (class C { })
* 枚举声明 (enum E { A, B, C })
* import 引用类型的声明

### Values

* let, const, var 声明
* namespace or module 包含值的声明
* enum 声明
* class 声明
* import 引用一个值的声明
* function 声明

### 内置组合

```ts
// ./foo.js
export var Bar: { a: Bar };
export interface Bar {
  count: number;
}
```

```ts
import { Bar } from "./foo";
let x: Bar = Bar.a;
console.log(x.count);
```

### 高级组合

```ts
interface Foo {
  x: number;
}
// ... elsewhere ...
interface Foo {
  y: number;
}
let a: Foo = ...;
console.log(a.x + a.y); // OK
```

### Adding using a namespace

命名空间声明可用于以任何不会产生冲突的方式添加新类型、值和命名空间。

```ts
class C {}
// ... elsewhere ...
namespace C {
  export let x: number;
}
let y = C.x; // OK
```
