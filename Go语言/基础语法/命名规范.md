# 命名规范

## Go 最独特的规则

- 在 Go 语言中，这不是一种“建议”或“习惯”，而是强制性的编译器规则
- 首字母大写（PascalCase）：表示 Public（公有）。可以被其他包（package）访问 例如：type User struct，func SaveData()
- 首字母小写（camelCase）：表示 Private（私有）。仅限当前包内部使用。例如：var dbConfig，func getCache()

## 包命名

- 必须是小写单数

## 变量与字段命名

- 驼峰命名法：Go 统一使用 MixedCaps 或 mixedCaps，禁用下划线。
- 缩写词全大写或全小写：当遇到缩写（如 HTTP, IP, ID）时，要保持大小写一致。

## 结构体与接口命名

- 结构体：通常为名词。
- 接口：惯例以 "er" 结尾

## 函数与方法命名

- 不要带上包名：如果你的包叫 user，函数就叫 Get() 而不是 GetUser()，因为外部调用是 user.Get()。
- Getter/Setter 规范
  - 获取值：直接用字段名，不要加 Get user.Name()
  - 设置值：用 Set 前缀 user.SetName("Go")

## 错误处理命名

- Error 变量：以 err 开头或 Err 开头（全局变量）。
  - 局部：if err := doSomething(); err != nil
  - 全局：var ErrNotFound = errors.New("not found")
