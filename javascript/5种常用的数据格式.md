# 5种常用的数据格式

## CSV (Comma-Separated Values)

- 特点：
  - 简单、紧凑的文本格式
  - 用逗号分隔值
  - 适合表格数据
  - 人类可读，易于编辑
- 使用场景：
  - 数据导入导出
  - 电子表格
  - 数据库备份
- 识别：
  - .csv        # 标准扩展名
  - .tsv        # Tab Separated Values (制表符分隔)
  - .txt        # 纯文本格式
  - .dat        # 数据文件

```csv
id,name,age,email,phone
1,张三,25,zhangsan@email.com,13800138001
2,李四,30,lisi@email.com,13800138002
3,王五,28,wangwu@email.com,13800138003
```

## JSON (JavaScript Object Notation)

- 特点：
  - 轻量级文本格式
  - 基于键值对
  - 支持数组和嵌套结构
  - 广泛用于API和数据交换
- 使用场景：
  - Web API
  - 配置文件
  - 数据存储
- 识别：
  - .json       # 标准扩展名
  - .js         # JavaScript文件中的JSON对象
  - .jsonc      # JSON with Comments (VS Code等)

```json
{
  "users": [
    {
      "id": 1,
      "name": "张三",
      "age": 25,
      "email": "zhangsan@email.com",
      "active": true,
      "tags": ["管理员", "VIP"],
      "profile": {
        "address": "北京市",
        "phone": "13800138001"
      }
    }
  ],
  "total": 2,
  "status": "success"
}
```

## XML (eXtensible Markup Language)

- 特点：
  - 标记语言，可自定义标签
  - 支持属性和元素
  - 严格的语法规则
  - 支持命名空间
- 使用场景：
  - 文档格式
  - 配置文件
  - Web服务
- 识别：
  - .xml        # 标准扩展名
  - .xsd        # XML Schema Definition
  - .xsl/.xslt  # XSL Transformations
  - .rss        # RSS feeds
  - .atom       # Atom feeds
  - .svg        # Scalable Vector Graphics

```xml
<?xml version="1.0" encoding="UTF-8"?>
<library>
    <book id="1">
        <title>编程之道</title>
        <author>
            <firstname>张</firstname>
            <lastname>伟</lastname>
        </author>
        <pages>300</pages>
        <price currency="CNY">59.99</price>
        <tags>
            <tag>编程</tag>
            <tag>技术</tag>
        </tags>
    </book>
    <book id="2">
        <title>数据结构与算法</title>
        <author>
            <firstname>李</firstname>
            <lastname>明</lastname>
        </author>
        <pages>450</pages>
        <price currency="CNY">89.99</price>
        <tags>
            <tag>算法</tag>
            <tag>数据结构</tag>
        </tags>
    </book>
</library>
```

## YAML (YAML Ain't Markup Language)

- 特点：
  - 人类可读的数据序列化标准
  - 使用缩进表示层级
  - 支持注释
  - 适合配置文件
- 使用场景：
  - 配置文件
  - 数据持久化
  - Docker配置
- 识别：
  - .yaml       # 推荐标准
  - .yml        # 简写形式

```yaml
# 应用配置
app:
  name: "MyApp"
  version: "1.0.0"
  debug: true
  
# 服务器配置
server:
  host: "localhost"
  port: 8080
  ssl:
    enabled: false
    cert_file: "server.crt"
    key_file: "server.key"
  
# 数据库配置
database:
  type: "postgresql"
  host: "localhost"
  port: 5432
  name: "myapp_db"
  credentials:
    username: "admin"
    password: "secret123"
    
# 用户列表
users:
  - id: 1
    name: "张三"
    email: "zhangsan@email.com"
    roles: ["admin", "user"]
    active: true
    
  - id: 2
    name: "李四"
    email: "lisi@email.com"
    roles: ["user"]
    active: false
    
# 简单列表
tags:
  - 开发
  - 测试
  - 部署
  
```

## TOML (Tom's Obvious, Minimal Language)

- 特点：
  - 配置文件格式
  - 语法类似INI
  - 支持数据类型
  - 比YAML更简洁
- 使用场景：
  - 应用程序配置
  - 包管理配置
  - 构建工具配置
- 识别：
  - .toml       # 标准扩展名
  - .ini        # 类似格式（INI files）

```toml
# 应用基本信息
title = "My Application"
version = "1.0.0"
description = "这是一个示例应用"

# 数据库配置
[database]
server = "localhost"
port = 5432
max_connections = 100
enabled = true

# 服务器配置
[server]
host = "0.0.0.0"
port = 8080
workers = 4
timeout = 30.0

# 日志配置
[logging]
level = "INFO"
file = "/var/log/app.log"
max_size = "100MB"
rotation = "daily"

# 用户配置
[[users]]
id = 1
username = "admin"
password = "hashed_password"
admin = true

[[users]]
id = 2
username = "user1"
password = "hashed_password2"
admin = false

# 应用特性
features = [
    "authentication",
    "authorization",
    "logging",
    "monitoring"
]
```
