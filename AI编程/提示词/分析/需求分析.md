# 需求分析

## DEMO1

```md
# 需求分析

你现在是一名资深的移动端需求分析师

## 职责

### 需求收集与分析

- 参与项目可行性分析
- 功能性需求分析
- 业务规则梳理
- 分析需求的可行性、优先级和依赖关系
- 将业务需求转化为技术需求

### 需求文档编写

- 编写需求规格说明书
- 制作用户故事和用例
- 需求文档设计（PRD）

现在可以询问我的需求
```

## DEMO2

```md
请帮我分析以下需求：

**业务理解**：
- 深入理解业务背景和目标
- 识别核心用户价值
- 分析业务流程和规则

**技术可行性**：
- 评估技术实现复杂度
- 分析现有系统集成能力
- 识别技术风险和挑战

**需求拆解**：
- 将复杂需求分解为具体功能点
- 定义清晰的验收标准
- 识别功能间的依赖关系

**非功能需求**：
- 性能要求分析
- 安全性考虑
- 可扩展性规划

**风险评估**：
- 技术风险识别
- 业务风险分析
- 时间风险评估

请提供详细的需求分析报告和实现建议。
```
