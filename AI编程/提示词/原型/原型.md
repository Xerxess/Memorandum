# 原型

```md
生活类App高保真原型设计项目提示词

项目概述

为生活类移动应用开发一套完整的高保真原型界面，确保设计方案可直接用于开发实施，并满足现代移动应用的用户体验标准。

工作流程

第一阶段：用户体验分析

- 深度分析生活类App的核心功能模块和用户使用场景
- 研究目标用户群体的行为模式和需求痛点
- 制定核心交互逻辑和用户操作流程
- 确定关键功能的优先级和页面权重

第二阶段：产品架构规划

作为产品经理角色，完成以下任务：
- 定义完整的信息架构和功能模块划分
- 设计用户操作路径和页面跳转逻辑
- 确定核心界面清单和页面层级关系
- 制定内容展示策略和数据结构

第三阶段：UI视觉设计

作为UI设计师角色，执行以下设计工作：
- 遵循iOS Human Interface Guidelines和Google Material Design规范
- 采用现代化设计语言，包括合适的色彩搭配、字体层级和间距规范
- 设计符合生活类应用特点的视觉风格
- 确保界面具备良好的可用性和视觉层次

第四阶段：高保真原型开发

使用Web技术栈实现交互式原型：
- 采用HTML5 + Tailwind CSS + FontAwesome图标库
- 实现响应式设计，确保多设备兼容性
- 添加必要的交互效果和动画过渡

技术规范要求

设备适配规范

- 目标设备：iPhone 15 Pro (393×852px, 设备像素比3.0)
- 视觉效果：圆角边框（border-radius: 20px），模拟真实手机外观
- 状态栏：iOS风格状态栏，包含时间、信号、电池等元素
- 导航栏：底部Tab Bar导航，符合iOS设计规范

代码结构要求

project/
├── index.html                 # 主入口文件
├── pages/                     # 页面文件夹
│   ├── home.html              # 首页
│   ├── profile.html           # 个人中心
│   ├── settings.html          # 设置页面
│   └── [其他页面].html
├── assets/                    # 资源文件夹
│   ├── css/
│   ├── js/
│   └── images/
└── components/                # 公共组件
    ├── navigation.html        # 导航组件
    └── statusbar.html         # 状态栏组件

界面展示方式

- index.html作为展示容器，使用iframe嵌入各页面
- 所有界面以网格布局平铺展示，便于整体预览
- 每个iframe尺寸设置为iPhone 15 Pro标准比例

图片资源标准

- 来源：Unsplash、Pexels等高质量图库，或Apple官方UI资源
- 格式：WebP优先，PNG/JPG备选
- 尺寸：至少2倍像素密度，确保高清显示
- 内容：与生活类应用场景相关的真实图片，避免占位符

浏览器兼容性

- 主要支持：Chrome 90+、Safari 14+、Firefox 88+
- 移动端：iOS Safari、Chrome Mobile

交付物清单

必要文件

1. index.html - 主展示页面
2. 独立页面文件 - 每个功能模块的完整页面
3. 样式文件 - 统一的CSS样式和主题配置
4. 资源文件 - 图片、图标等静态资源
5. 说明文档 - 页面功能说明和使用指南

界面覆盖范围

- 启动页/引导页
- 主界面/仪表板
- 功能模块页面（根据具体需求定制）
- 个人中心/设置页面
- 表单填写/数据输入页面
- 列表/详情页面组合

验收标准

视觉质量标准

- 界面美观度达到App Store发布标准
- 色彩搭配和谐，符合生活类应用调性
- 字体层级清晰，信息易读性良好
- 图标和按钮设计统一，交互反馈明确

技术实现标准

- 代码结构清晰，注释完整
- 响应式设计完善，多设备适配良好
- 加载速度快，用户体验流畅
- 无明显的浏览器兼容性问题

功能完整性标准

- 覆盖应用核心功能的所有界面
- 页面间逻辑关系清晰合理
- 关键交互流程完整可用
- 界面内容丰富真实，具备实用性

```
